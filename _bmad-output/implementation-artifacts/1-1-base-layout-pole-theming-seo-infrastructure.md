# Story 1.1: Base Layout, Pole Theming & SEO Infrastructure

Status: review

## Story

As a visitor,
I want every page to have a consistent layout with visual identity that shifts per pole and proper SEO metadata,
so that I always know which section I'm in and every page is discoverable by search engines.

## Acceptance Criteria

1. Every page renders within BaseLayout with header, main content area, and footer using consistent spacing, typography (Montserrat), and max-width 1280px
2. Pole pages (restaurant/aventure/evenements) apply accent color, light background, and CTA labels resolved from centralized `pole-config.ts` via inline `style` attributes
3. Every page includes unique `<title>`, `<meta name="description">`, canonical URL, Open Graph tags, and hreflang tags (FR/EN/ES) via SEOHead.astro
4. Homepage emits LocalBusiness JSON-LD, restaurant pages emit Restaurant JSON-LD, activity pages emit SportsActivityLocation JSON-LD — all generated by `src/lib/seo.ts`
5. An XML sitemap is auto-generated with all pages and proper hreflang references
6. Astro config output mode is set to `hybrid` for static pages + SSR Keystatic admin
7. Montserrat fonts are self-hosted from `public/fonts/` with `font-display: swap` and preloaded critical weights
8. A `.env.example` documents all required environment variables

## Dev Notes

### Architecture Patterns (MUST follow)

- **Pole-aware pattern:** Components accept `pole` prop typed as `'restaurant' | 'aventure' | 'evenements' | null`. Colors applied via inline `style` attributes, NEVER dynamic Tailwind classes. [Source: architecture.md#Pole-Aware Component Pattern]
- **Import aliases:** Always use `@/` prefix for imports from `src/`. [Source: architecture.md#Import Patterns]
- **Tailwind v4:** Config via `@theme` directive in `src/styles/global.css`. No `tailwind.config.js`. [Source: CLAUDE.md]
- **i18n utilities:** Use `getLangFromUrl(Astro.url)` in every page frontmatter. Use `getLocalizedPath()` for all links. [Source: architecture.md#i18n Patterns]
- **Rendering boundary:** SEOHead and BaseLayout are Astro components (zero JS). No Svelte islands needed for this story. [Source: architecture.md#Rendering Boundary]
